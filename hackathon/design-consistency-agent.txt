WHY THIS EXISTS
---------------
The Kaimi web dashboard (the Go app in internal/dashboard) was built against a
locked design handoff, but the screens have drifted: pages re-hardcode colors /
spacing / type instead of consuming the ported design-system tokens & components,
the designed fonts (Figtree / Geist Mono) fall back to system fonts so it doesn't
match the comps, and some shared components aren't rendering correctly across all
their states. This agent's job is to make the whole web UI visually consistent
with the design system — audited and verified IN A REAL BROWSER, one surface at a
time — and to confirm every shared component works across every state.

HARD RULES (do not break)
-------------------------
- DEFINE ONCE, REUSE: every page consumes StyleTag() + token CSS variables +
  components.go helpers. NEVER re-hardcode hex colors, spacing, radii, shadows, or
  component markup per page. Missing component -> add it to components.go (with a
  test) and reuse it.
- NO external assets/fonts. Inline SVG and base64 data-URIs are OK; Google Fonts /
  external CSS / network fonts are NOT. To make the designed fonts actually render,
  self-host Figtree + Geist Mono as @font-face base64 data-URIs inside the embedded
  CSS; otherwise keep a consistently-applied system fallback. Document the choice in
  ux-spec.md either way.
- Amber #E8870E on #FFF3E0 is reserved app-wide for "a human is needed" — never
  decorative.
- Follow CLAUDE.md / WORKFLOW.md: ticketed work with approved acceptance criteria,
  TDD, `make all` green before any PR, one PR per surface (base main), and NEVER
  merge your own code — a human (Malik/Timm) merges.
